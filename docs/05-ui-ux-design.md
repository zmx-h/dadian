# 搭电 (Dā Diàn) — UI/UX 设计规范

> 版本: v1.6 | 日期: 2026-06-21 | 平台: Web + H5 | 框架: Vue 3 | 动效: @vueuse/motion

---

## 1. 设计哲学

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **驾驶舱感** | 用户不是在「使用 App」，而是在「操控飞船」。界面有科技感但不过度 |
| **玻璃与光** | 毛玻璃材质 + 琥珀光晕是品牌视觉核心，营造赛博浪漫氛围 |
| **内容即界面** | 尽量去掉 chrome（边框、背景、分割线），用模糊和光区分层级 |
| **舒适暗色** | 全站暗色基调（stone-950 底色），但不是纯黑，留出温暖感和深度 |
| **会呼吸的动效** | 弹簧物理曲线 > 线性缓动。界面有生命感，不是冰冷的过渡 |

### 1.2 设计反模式（禁止）

- ❌ 弹出式 Modal 确认框 — 改为内联展开确认
- ❌ 尖锐的 90° 直角边框 — 最小 8px border-radius
- ❌ 高饱和度大面积色块 — 只用暗色底 + 琥珀色点缀
- ❌ 「加载中」骨架屏 — 用微光呼吸动画代替
- ❌ 表单验证红色错误框 — 用琥珀虚线边框 + 温和提示文案

## 3. 字体系统

原设计字体适用于全平台：

| 角色 | 字体 | 回退链 | 声明 |
|------|------|------|------|
| 标题 | Space Grotesk | → Noto Sans SC → system-ui sans-serif | 西文 Space Grotesk，中文回退至思源黑体 |
| 正文 | DM Sans | → Noto Sans SC → system-ui sans-serif | 同上 |
| 数字/代码 | IBM Plex Mono | → monospace | 等宽数字 |

**中文回退说明**：Space Grotesk 和 DM Sans **不含 CJK 字形**。所有中文文本会沿 fallback 链下落到 Noto Sans SC（思源黑体）。Noto Sans SC 与 Space Grotesk 的 x-height 和 stroke weight 经过刻意匹配，中西文混排视觉一致性可接受。如果 CDN 未加载到 Noto Sans SC（用户网络极差），最终回退到 system-ui（Windows: 微软雅黑 / macOS: PingFang SC），此时存在 ~20% x-height 差异，属降级容忍。

**Web 加载（自托管 + CDN，不使用 Google Fonts）：**
Google Fonts 在中国大陆延迟 500ms-3s，会产生 FOUT（无样式文字闪烁），对「驾驶舱沉浸感」是毁灭性的。

解决方案：
```css
/* 从自托管 CDN 加载 woff2，由 OSS + CDN 分发 */
@font-face {
  font-family: 'Space Grotesk';
  src: url('/fonts/SpaceGrotesk.woff2') format('woff2');
  font-weight: 400 700;
  font-display: swap;
}
@font-face {
  font-family: 'DM Sans';
  src: url('/fonts/DMSans.woff2') format('woff2');
  font-weight: 400 700;
  font-display: swap;
}
@font-face {
  font-family: 'IBM Plex Mono';
  src: url('/fonts/IBMPlexMono.woff2') format('woff2');
  font-weight: 400 500;
  font-display: swap;
}
```

备选方案：fontsource cn 镜像或中文网字计划的国内节点。

### 8.3 三个核心页面的桌面布局

**出发页渐进式承诺流程（Web 适配）：**

产品策略已从"命运骰子粗暴决定"改为"渐进式承诺 + 灵感骰子兜底"。出发页交互分为三步：

```
Step 1: 选区域（粗粒度）
  → 地图缩放到城市范围，高亮推荐区域（基于用户画像 + 历史偏好）
  → 点击某个区域（如"武康路-安福路片区"）

Step 2: 选地点（细粒度）
  → 该区域内的 POI 卡片流，按<电量匹配度>排序
  → 可浏览、筛选、点开详情

Step 3: 定路线（自动）
  → 选定地点后，系统自动规划路线
  → "还犹豫吗？试试灵感骰子"（骰子按钮以毛玻璃胶囊出现在确认区右侧，作为兜底加速器而非主路径）
```

## 9. 图标系统

使用 Lucide Icons（线性图标，stroke 表现优秀）：

```
导航:    ⚡ zap (出发), 🧭 compass (探索), 📷 camera (回忆), 👥 users (社区)
职业:    🕵️ search (特工), 🍜 utensils (美食), 👻 coffee (NPC)
操作:    🎲 dice-5 (骰子), 🚀 rocket (出发), ⏸️ pause (暂停), 🔄 refresh-cw (转职)
状态:    ● 灰色(离线), ● 琥珀呼吸(出行中)
```

**关键操作必须图标+颜色双通道冗余。** 琥珀色（`#f59e0b`）在暗色背景下对红色盲（protanopia）用户对比度可能不满足 WCAG 标准。「出发」「骰子」等关键按钮不能只靠颜色区分——必须配图标或形状差异。

**3D Tilt 触屏冲突处理**：Surface Pro 等触屏笔记本上 `mousemove` 和 `touch` 事件会同时触发——用户手指滑动翻页时照片也会倾斜，体验混乱。解决方案：监听 `pointerenter`/`pointerleave` 事件，使用 `event.pointerType === 'touch'` 判定；当 pointer type 为 touch 时，禁用 Tilt 效果，仅保留翻页手势。

## 11. 原设计问题修正

| 原设计 | 问题 | 修正 |
|--------|------|------|
| 纯移动端 Dock 设计 | Web 端不适用 | 桌面端改用垂直导航条 + 顶部标签 |
| 无桌面端布局 | 浪费大屏空间 | 增加三列布局、侧边任务面板 |
| 毛玻璃 CSS 写死 Tailwind 类 | 不可维护 | 提取为 `.glass-*` 工具类 |
| 响应式只提二词 | 不够具体 | 补充完整断点 + 布局 |
| 无组件库分层 | 组件复用性差 | 建立 glass-card/glass-button 基础组件 |
| 无输入框/表单设计 | 欢迎流程无法落地 | 补充赛博输入框 + 验证码样式 |
| 无桌面交互说明 | hover/右键无定义 | 增加桌面端专属交互描述 |
| 分享图无尺寸规范 | 图片规格缺失 | 1080×1350px 竖版 + 16:9 横版选项 |
